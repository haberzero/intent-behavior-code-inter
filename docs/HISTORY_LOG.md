# HISTORY_LOG（历史归档）

> 本文件用于保存已完成工作的详细演进记录、阶段性日志、历史方案与过时任务过程。
> `docs/COMPLETED.md` 仅保留精简里程碑，避免持续膨胀。
>
> 最后更新：2026-05-08

---

## 归档原则

1. 已完成且不再作为当前决策依据的详细日志，统一迁入本文件。
2. 历史术语（如旧类型模型名、旧阶段基线）允许保留，但需显式标注为“历史语境”。
3. 当前状态、当前主线、当前优先级，以以下文档为准：
   - `docs/NEXT_STEPS.md`
   - `docs/PENDING_TASKS.md`
   - `docs/TYPE_SYSTEM_DESIGN.md`
   - `docs/VM_INTERPRETER_DESIGN.md`
